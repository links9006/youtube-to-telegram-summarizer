# Security Policy

## Supported Versions

Only the latest release on the `main` branch receives security updates.

## Reporting a Vulnerability

If you discover a security vulnerability, **please do not open a public GitHub issue**.

Instead, report it privately:

- Use **GitHub Security Advisories** → *"Report a vulnerability"* on the repo's Security tab, **or**
- Email the maintainer directly.

Please include:
- A clear description of the issue and its potential impact.
- Steps to reproduce (proof of concept).
- Suggested fix, if you have one.

We will acknowledge receipt within **72 hours** and aim to ship a fix within **14 days** for high-severity issues.

## Responsible Disclosure

We kindly ask that you:
- Give us reasonable time to investigate and fix the issue before any public disclosure.
- Avoid accessing or destroying other users' data.
- Do not degrade service availability while testing.

## Scope

In scope: the code in this repository, its default configuration, and the documented deployment flow.

Out of scope: vulnerabilities in third-party services (Telegram, YouTube, OpenRouter, OpenAI/Whisper, Hugging Face) — report those to the respective vendors.
